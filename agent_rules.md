# エージェント自動運用ルール

## 1. サーバー再起動時の自動スケジュール再設定
*   **事象**: システムメンテナンスやサーバー再起動が発生すると、登録されていたスケジュール（Cronタスク）がリセットされます。
*   **対応**: エージェントが起動した際に、サーバー再起動の通知（`[Notice] All your subagents...` などのメッセージ）を受信した場合、または設定されていたスケジュールが消失していることを検知した場合、エージェントはユーザーの指示を待たずに自発的かつ速やかに「毎日AM9時」の競合環境調査タスクを `schedule` ツールを用いて再設定してください。
    *   **設定内容**:
        *   `CronExpression`: `0 9 * * *`

        *   `Prompt`: `report_instruction.mdの指示に従って、いかりスーパーの周辺競合環境調査を実行し、レポートを作成してください。`

## 2. 変更時の即時 Git コミット
*   **ルール**: ファイル構成（ファイルの新規作成・削除・リネーム）や、既存ファイルの内容、ソースコードに変更を加えた場合は、その都度（変更を行ったターン内で）必ず `git commit` を実行してください。
    *   コミットメッセージは具体的で、変更内容が分かりやすいように記述してください（例: `feat: ...`, `fix: ...`, `refactor: ...`）。
    *   これにより、要件変更やリファクタリングの過程で変更履歴が失われるのを防ぎ、いつでも過去のバージョンに復元できるようにします。
